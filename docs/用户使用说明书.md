## 系统概述

本系统是一个完整的餐饮点餐管理平台，支持多店铺管理、多角色协作、实时订单推送等功能。

### 核心功能

- ✅ 扫码点餐（顾客端）
- ✅ 多角色登录（店长、厨师、传菜员、收银员）
- ✅ 实时订单推送（WebSocket）
- ✅ 菜单管理
- ✅ 库存管理
- ✅ 会员管理
- ✅ 营收统计

## 顾客端使用说明

### 访问点餐页面

```
http://129.226.196.76/customer_order_v3.html
```

### 点餐流程

#### 步骤1：选择桌号
- 在桌号列表中选择对应的桌号
- 或扫描桌上的二维码自动识别

#### 步骤2：浏览菜单
- 菜单按分类展示（热菜、凉菜、主食、饮品等）
- 点击菜品可以查看详细信息
- 可以搜索菜品名称

#### 步骤3：加入购物车
- 点击菜品旁的"+"按钮加入购物车
- 可以调整菜品数量
- 在购物车中可以删除不需要的菜品

#### 步骤4：提交订单
- 确认订单信息（桌号、菜品、金额）
- 可选填写会员手机号
- 点击"提交订单"按钮

#### 步骤5：等待制作
- 实时查看订单状态
- 订单状态：
  - `待确认` - 订单已提交，等待店员确认
  - `制作中` - 厨师正在制作
  - `待传菜` - 菜品制作完成，等待传菜
  - `上菜中` - 正在上菜
  - `已完成` - 全部菜品已上完

### 会员功能

- 输入手机号可累积积分
- 会员可以享受折扣优惠
- 查看消费记录和积分余额

## 常见问题

### Q1: 找不到登录页面？

**A:** 访问 http://129.226.196.76/login.html

### Q2: 忘记密码怎么办？

**A:** 联系系统管理员重置密码。默认密码请参考账号体系说明。

### Q3: 顾客下单后，厨房看不到订单？

**A:** 检查以下几点：
1. 确认WebSocket服务是否正常运行
2. 刷新厨房显示页面
3. 检查网络连接

### Q4: 如何添加新菜品？

**A:**
1. 以店长身份登录
2. 进入菜单管理页面
3. 点击"添加菜品"
4. 填写菜品信息并保存

### Q5: 如何生成桌号二维码？

**A:**
1. 访问店铺设置页面
2. 在桌号管理中选择要生成二维码的桌号
3. 点击"生成二维码"
4. 下载并打印

### Q6: 订单状态不更新？

**A:** 检查WebSocket连接状态，刷新页面或重新登录。

### Q7: 会员积分如何累积？

**A:** 顾客点餐时输入手机号，系统自动累积积分。消费1元=1积分。

### Q8: 如何查看营收数据？

**A:**
1. 以店长或总公司身份登录
2. 在工作人员端首页可以查看营收统计
3. 查看详细的销售报表

## 系统架构

### 技术栈

- **前端**: HTML5, CSS3, JavaScript, Vue.js, Element Plus
- **后端**: Python 3.10, FastAPI, Uvicorn
- **数据库**: PostgreSQL 14
- **实时通信**: WebSocket
- **Web服务器**: Nginx
- **部署**: 腾讯云 Ubuntu 服务器

### 服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx | 80 | Web服务器和反向代理 |
| 后端API | 8000 | RESTful API服务 |
| WebSocket | 8001 | 实时通信服务 |

### 目录结构

```

├── frontend/              # 前端页面
│   ├── login.html         # 登录页

│   └── ...

    └── ...
```
